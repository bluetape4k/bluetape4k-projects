## 🔍 주요 내용 요약

### 모듈 역할

- **목표**: 로컬 JVM에서 리더(Leader) 선출 및 동시 실행 제어
- **범위**: 동기, 비동기, Virtual Thread, Coroutines
- **제약**: 분산 환경 미지원 (단일 JVM만)

### 핵심 구성

- **인터페이스**: 7개 (동기, 비동기, Virtual Thread, Coroutines)
- **구현체**: 6개 (모두 local/ 패키지)
- **기술**: ReentrantLock, Semaphore, Mutex, Virtual Threads
- **의존성**: 외부 라이브러리 0개

### 중요 포인트

✅ **장점**:

- 단순하고 명확한 설계
- 3가지 동시성 모델 지원 (Thread, Virtual Thread, Coroutine)
- 안전한 자원 관리 (try-finally)
- 확장 가능한 구조

❌ **주의사항**:

- 분산 환경 미지원
- Coroutines Mutex는 재진입 불가
- Java 21+ (Virtual Thread 위해)
- 상태 조회는 근사값

## 💬 질문과 답변

### Q1: 어떤 구현체를 선택할까?

**A**: 우선순위 결정 트리:

```
1. Coroutines 사용 중?
   → LocalSuspendLeaderElection 또는 LocalSuspendLeaderGroupElection

2. Virtual Thread 활용 (Java 21+)?
   → LocalVirtualThreadLeaderElection

3. CompletableFuture 필요?
   → LocalLeaderElection 또는 LocalAsyncLeaderElection

4. 복수 동시 실행 제한?
   → LocalLeaderGroupElection

5. 기본값?
   → LocalLeaderElection (모든 경우 커버)
```

자세한 가이드는 **SUMMARY.md**의 "사용 시나리오" 참고

---

### Q2: 재진입(reentrancy)을 지원하나요?

**A**: 구현체별로 다릅니다:

| 구현체                              | 재진입           |
|----------------------------------|---------------|
| LocalLeaderElection              | ✅ 지원          |
| LocalVirtualThreadLeaderElection | ✅ 지원          |
| LocalSuspendLeaderElection       | ❌ 미지원 (Mutex) |
| LocalLeaderGroupElection         | ⚠️ 주의 (슬롯 소진) |

자세한 설명은 **ANALYSIS.md**의 "주의사항" 참고

---

### Q3: 분산 환경에서 사용 가능한가요?

**A**: 현재 **불가능**합니다. 모든 구현체는 단일 JVM 내에서만 작동합니다.

분산 환경이 필요하면:

- ZooKeeper 기반 구현 추가
- Redis 기반 구현 추가
- Hazelcast 기반 구현 추가

가능한 구조는 **FILE-INDEX.md**의 "확장 포인트" 참고
